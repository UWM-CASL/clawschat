ClawsChat

ClawsChat Help

ClawsChat is a student-focused local AI chat app. It runs supported models in your browser, keeps your work on your device by default, and gives you tools to compare, revise, stop, copy, and export conversations for study.

Open chat home Project README

Student quick note If you just want to get started: open a conversation, pick a model, type a prompt, and send it. The model loads when you send your first message.

Start a conversation
  1. Select Start a conversation from the home screen.
  2. Choose a model in the pre-chat panel.
  3. Type your first prompt in Message input.
  4. Send the message to begin model loading and response generation.
  5. Watch the loading progress area until the transcript starts streaming.

ClawsChat loads the selected model only when you send the first message for that conversation.

Chat, stop, and review responses
  • Responses stream into the transcript as they are generated.
  • Use Stop generating at any time to cancel the current reply.
  • After a reply finishes, you can copy it, regenerate it, or run Fix from the focused transcript shortcut.
  • If a model exposes thinking content, ClawsChat places it in a separate Thinking section instead of mixing it into the final answer.
  • Markdown and LaTeX math render directly in the transcript.

Fix creates a revised answer by running a critique, revision, and validation flow on the current response.

Manage conversations, branches, and variants
  • Conversations are saved locally and restored when you reopen the app.
  • Settings → Conversation → Export downloads a zip with JSON, Markdown, and stored artifacts for every saved conversation.
  • Settings → Conversation → Delete Conversations removes all saved conversations and their stored artifacts from this browser.
  • New conversations start untitled and rename automatically after the first full reply.
  • Use Edit to replace a user message and continue from that point.
  • Use Branch to create an alternate path without deleting the original branch.
  • Use Regenerate to create multiple model replies for the same turn and compare them with the left and right variant controls.
  • After the first completed reply, the conversation menu can export the visible branch as Markdown or JSON.
Use settings with confidence
  • Backend lets you choose Auto, WebGPU only, or WASM only.
  • Theme lets the app match light, dark, or system appearance.
  • Maximum output tokens controls reply length.
  • Context size controls how much short-term conversation memory is used.
  • Temperature, Top K, and Top P adjust response style.
  • Generation settings are model-aware, saved per model, and disabled until the model is loaded.
  • Default system prompt applies only to newly created conversations.
  • Transcript view switches between standard and compact spacing.

Accessibility and focus behavior

Accessibility and focus behavior

  • Hidden skip links appear on keyboard focus so you can jump straight to help topics, keyboard shortcuts, or accessibility guidance.
  • All primary controls can be reached with keyboard navigation.
  • Dialogs and panels close with Escape and return focus sensibly.
  • The transcript has clear speaker labels for user and model messages.
  • Status updates such as loading, generating, stopped, and errors use dedicated announcements instead of reading every streamed token.
  • Transcript helper controls help you move to the start, the end, or the message input.
Privacy, storage, and exports
  • Prompts and outputs stay in the browser unless you export them yourself.
  • ClawsChat stores conversations, settings, and model choices locally in browser storage.
  • Downloaded model files are cached for reuse in the browser.
  • JSON exports preserve structured conversation data for the visible branch.
  • Markdown exports provide a readable version of the visible branch.
Troubleshooting
  • If WebGPU fails, switch the backend to WASM only in Settings and try again.
  • If generation seems stuck, use Stop generating and send the prompt again.
  • Token and sampling fields stay disabled until the model has fully loaded.
  • If transcript shortcuts are not working, focus a transcript message and confirm Enable single-key transcript shortcuts is on.
  • Use Debug info in Settings when you need engine or initialization details.
